뭐했지 (DidI)

최종 수정일:

1. 데이터 처리 원칙

"뭐했지 (DidI)" 앱은 사용자의 프라이버시를 최우선으로 생각합니다. 본 서비스는 사용자의 일상 데이터를 기기 내부에서 처리하는 '로컬 우선(Local-first)' 철학을 유지하며, AI 기능을 포함한 모든 서비스에서 개인을 식별할 수 있는 정보를 외부에 무단으로 노출하지 않는 것을 절대 원칙으로 합니다.

앱을 통해 기록되는 상세 위치와 앱 사용 기록은 기술적으로 보호되며, AI 분석 등 서버 처리가 필요한 경우에도 개인 장소명을 제거하는 등 철저한 비식별화 과정을 거칩니다.

2. 수집 및 활용하는 데이터 항목

앱의 주요 기능인 '자동 활동 기록'을 위해 다음과 같은 정보에 접근하고 처리합니다.

A. 기기 내 로컬 처리 데이터 (서버 전송 없음)

  • 위치 정보 (Background Location): 하루 이동 경로 및 머무른 장소를 파악하여 타임라인을 구성하기 위해 사용됩니다.
  • 앱 사용 통계 (Usage Stats): 특정 시간에 어떤 앱을 사용했는지 분석하여 활동(예: 공부, SNS, 영상 시청)을 추론합니다.
  • 활동 인식 (Activity Recognition): 사용자의 이동 상태(걷기, 차량 이동, 정지)를 감지하여 기록의 정확도를 높입니다.

B. 선택적 클라우드 동기화 데이터 (사용자 선택 시)

기기 변경이나 데이터 백업을 위해 사용자가 '클라우드 동기화'를 활성화할 경우에만 다음 데이터를 서버에 보관합니다.

  • 암호화된 활동 기록: 타임라인 데이터 및 사용자 설정 정보. 모든 데이터는 **군사 수준의 AES-256 암호화**를 거쳐 저장되며, 본인 외에는 누구도 해독할 수 없습니다.
  • 소셜 로그인 식별자: Google 계정을 통한 사용자 인증 정보 (본인 확인 및 데이터 소유권 검증용).

C. AI 데일리 인사이트 처리 데이터 (사용자 활성 시)

사용자가 'AI 데일리 인사이트' 기능을 켤 경우, 매일 밤 하루의 요약을 위해 다음 데이터를 일시적으로 서버에서 처리합니다.

  • 비식별화된 활동 요약: 시간대별 활동 유형과 장소 카테고리(예: 집, 회사, 카페) 정보. ※ 사용자가 직접 설정한 고유 지점명이나 상세 주소는 전송 전 모두 제거됩니다.
  • 일기 텍스트: 사용자가 당일 작성한 일기 내용(선택 사항).
  • 데이터 보호 및 파기: 전송된 데이터는 AI 모델의 학습(Training)에 절대 활용되지 않으며, 인사이트 생성 즉시 서버 메모리에서 소멸됩니다. (OpenAI API 정책에 따라 보안 모니터링을 위해 암호화된 상태로 30일간 보관 후 영구 파기)

D. 서비스 개선을 위한 익명 통계

  • 익명 기능 선호도: 사용자가 어떤 인사이트 스타일(감성형/이성형)을 선호하는지에 대한 단순 카운트 데이터를 수집합니다. 해당 정보는 개인을 식별하거나 다른 데이터와 결합될 수 없는 완전한 익명 통계로 관리됩니다.

3. 특수 권한 사용 안내

본 앱은 자동 기록 기능을 위해 안드로이드 시스템의 다음 권한을 요청합니다.

  • 백그라운드 위치 액세스: 앱이 화면에 떠 있지 않을 때도 위치 변화를 감지하여 끊김 없는 타임라인을 구성하기 위해 필수적입니다.
  • 사용 정보 액세스: 앱별 사용 시간을 쿼리하여 현재 사용자가 무엇을 하고 있는지 지능적으로 추론하기 위해 사용됩니다.

4. 데이터 보관 및 파기

로컬 데이터는 앱 삭제 시 함께 제거됩니다. 클라우드 동기화를 사용 중인 경우, 앱 내 '설정 > 계정 관리' 메뉴의 [계정 삭제] 기능을 통해 서버에 보관된 모든 암호화 데이터를 즉시 영구 삭제할 수 있습니다.

5. 웹을 통한 계정 및 데이터 삭제 요청

앱을 이미 삭제했거나 앱에 접속할 수 없는 경우, 아래의 방법으로 데이터 삭제를 요청하실 수 있습니다.

  • 요청 방법: 지원 이메일로 가입 시 사용한 Google 계정 정보를 기재하여 발송
  • 이메일 주소: support@franholdings.cloud
  • 처리 기간: 접수 후 영업일 기준 7일 이내 데이터 파기 및 회신

6. 제3자 서비스 제공

안정적인 서비스 제공 및 고품질 AI 인사이트 생성을 위해 아래 외부 서비스를 활용합니다. 각 서비스는 엄격한 데이터 보호 표준을 준수합니다.

  • Google Firebase: 사용자 인증(Auth), 암호화된 데이터 보관(Firestore) 및 익명 통계 수집(Analytics)을 위한 인프라 제공.
  • OpenAI: AI 데일리 인사이트 메시지 생성을 위한 인공지능 엔진 제공. (전송된 데이터는 모델 학습에 사용되지 않음)

1. Principles of Data Processing

"DidI" app prioritizes your privacy above all else. We adhere to a 'Local-first' philosophy, processing your daily life data directly on your device. Our core principle is never to expose personally identifiable information to external parties for AI features or any other services.

Detailed location and app usage records are technically protected. Even when server processing is required (e.g., for AI analysis), data undergoes strict de-identification, including the removal of specific place names.

2. Data Items Collected and Used

To provide the core 'Automatic Activity Logging' feature, we access and process the following information:

A. Locally Processed Data (On-device, No Server Transmission)

  • Background Location: Used to identify daily movement paths and stayed locations to construct your timeline.
  • Usage Stats: Analyzes which apps were used at specific times to infer activities (e.g., studying, SNS, watching videos).
  • Activity Recognition: Detects your movement state (walking, driving, stationary) to improve log accuracy.

B. Optional Cloud Synchronization Data (User-selected)

Only if you enable 'Cloud Sync' for device changes or data backup, the following data is stored on our servers:

  • Encrypted Activity Records: Timeline data and user settings. All data is stored with **Military-grade AES-256 encryption**, and no one except the user can decrypt it.
  • Social Login Identifiers: Authentication info via Google (for identity verification and data ownership validation).

C. AI Daily Insight Processing Data (User-activated)

If you enable the 'AI Daily Insight' feature, the following data is temporarily processed on our servers for nightly summaries:

  • De-identified Activity Summary: Activity types and place categories (e.g., home, office, cafe) by time of day. ※ All specific place names or detailed addresses are removed before transmission.
  • Diary Text: Optional diary entries written by the user for that day.
  • Protection and Deletion: Transmitted data is NEVER used to train AI models and is destroyed from server memory immediately after insight generation. (Encrypted for 30 days for security monitoring per OpenAI policy, then permanently destroyed).

D. Anonymous Statistics for Service Improvement

  • Feature Preferences: Simple counts of which insight styles (Emotional/Rational) are preferred. This is managed as purely anonymous statistics that cannot identify individuals.

3. Special Permissions Information

This app requests the following Android system permissions for automatic logging:

  • Background Location Access: Essential for detecting location changes even when the app is not on screen to build a seamless timeline.
  • Usage Information Access: Used to intelligently infer what the user is doing by querying app usage time.

4. Legal Basis for Processing (EEA Users)

For users in the European Economic Area (EEA), we process data based on the following legal grounds under the GDPR:

  • Consent: For cloud synchronization and AI-driven insights.
  • Contractual Necessity: To provide the core automatic logging and timeline services.

5. Data Subject Rights (GDPR)

You have the following rights regarding your personal data:

  • Right to Access & Portability: You can request a copy of your data through app settings.
  • Right to Rectification: You can correct any inaccurate information within the app.
  • Right to Erasure ('Right to be Forgotten'): You can delete all data via the 'Delete Account' menu.
  • Right to Lodge a Complaint: You have the right to complain to your local Data Protection Authority.

6. International Data Transfers

We use **Google Firebase** and **OpenAI**, whose servers are located in the United States. We ensure that your data is protected using Standard Contractual Clauses (SCCs) and other appropriate safeguards for international transfers.

7. Data Retention and Destruction

Local data is removed when the app is uninstalled. For cloud sync users, you can permanently delete all encrypted data on the server via the 'Settings > Account Management > Delete Account' menu within the app.

8. Requesting Account and Data Deletion via Web

If you have already deleted the app or cannot access it, you can request data deletion using the method below:

  • Method: Email your request with the Google account used during registration.
  • Email: support@franholdings.cloud
  • Processing Time: Data destruction and confirmation within 7 business days.

9. Third-party Service Providers

To provide stable services and high-quality AI insights, we utilize the following external services, each adhering to strict data protection standards:

  • Google Firebase: Provides infrastructure for authentication (Auth), encrypted storage (Firestore), and anonymous statistics (Analytics).
  • OpenAI: Provides the AI engine for generating daily insight messages. (Transmitted data is NOT used for model training).

10. Contact Information

Support Team: support@franholdings.cloud

1. データ処理の原則

「なにやったっけ (DidI)」アプリは、お客様のプライバシーを最優先に考えています。本サービスは、日常データを端末内で処理する「ローカルファースト」の哲学に基づき、AI機能を含むすべてのサービスにおいて、個人を特定できる情報を外部に無断で露出しないことを絶対原則としています。

詳細な位置情報やアプリの使用履歴は技術的に保護されており、AI分析などのサーバー処理が必要な場合でも、特定の場所名を除去するなど、徹底した非識別化プロセスを経ています。

2. 収集および活用するデータ項目

アプリの主要機能である「自動活動記録」を提供するため、以下の情報にアクセスし、処理を行います。

A. 端末内のローカル処理データ（サーバー送信なし）

  • 位置情報 (Background Location): 一日の移動経路および滞在場所を把握し、タイムラインを構成するために使用されます。
  • アプリ使用統計 (Usage Stats): 特定の時刻にどのアプリを使用していたかを分析し、活動（例：勉強、SNS、動画視聴など）を推論します。
  • 活動認識 (Activity Recognition): お客様の移動状態（徒歩、車移動、停止）を検知し、記録の正確度を高めます。

B. オプションのクラウド同期データ（ユーザー選択時）

機種変更やデータバックアップのために「クラウド同期」を有効にした場合に限り、以下のデータをサーバーに保管します。

  • 暗号化された活動記録: タイムラインデータおよびユーザー設定情報。すべてのデータは**軍事レベルのAES-256暗号化**を施して保存され、ユーザー本人以外は解読できません。
  • ソーシャルログイン識別子: Googleアカウントによる認証情報（本人確認およびデータの所有権確認用）。

C. AI デイリーインサイト処理データ（ユーザー有効時）

「AI デイリーインサイト」機能を有効にした場合、毎晩の一日の要約のために以下のデータを一時的にサーバーで処理します。

  • 非識別化された活動概要: 時間帯別の活動タイプと場所カテゴリ（例：自宅、会社、カフェ）情報。※ ユーザーが設定した固有の地点名や詳細な住所は、送信前にすべて削除されます。
  • 日記テキスト: お客様がその日に作成した日記の内容（任意）。
  • データ保護および破棄: 送信されたデータはAIモデルの学習（Training）には一切使用されず、インサイト生成後、即座にサーバーのメモリから消去されます。（OpenAIのポリシーに基づき、セキュリティ監視のために暗号化された状態で30日間保管された後、永久に破棄されます）。

D. サービス改善のための匿名統計

  • 機能設定の好み: どのインサイトスタイル（感情型/論理型）が好まれているかなどの単純なカウントデータ。個人を特定できない完全な匿名統計として管理されます。

3. 特殊権限の使用案内

本アプリは、自動記録機能のためにAndroidシステムの以下の権限をリクエストします。

  • バックグラウンド位置情報の利用: アプリが画面に表示されていないときでも位置の変化を検知し、途切れのないタイムラインを構築するために不可欠です。
  • 使用状況へのアクセス: アプリごとの使用時間をクエリし、現在お客様が何をしているかをインテリジェントに推論するために使用されます。

4. データの保管と破棄

ローカルデータはアプリ削除時に同時に除去されます。クラウド同期を使用している場合は、アプリ内の「設定 > アカウント管理 > アカウント削除」機能を通じて、サーバーに保管されたすべての暗号化データを即座に永久削除できます。

5. Webを通じたアカウントおよびデータの削除リクエスト

アプリをすでに削除した場合やアプリにアクセスできない場合は、以下の方法でデータの削除をリクエストできます。

  • リクエスト方法: サポートメールに、登録時に使用したGoogleアカウント情報を記載して送信
  • メールアドレス: support@franholdings.cloud
  • 処理期間: 受付後、営業日基準で7日以内にデータを破棄し、回答いたします

6. 第三者サービスの提供

安定したサービスの提供および高品質なAIインサイト生成のために、以下の外部サービスを活用しています。各サービスは厳格なデータ保護基準を遵守しています。

  • Google Firebase: ユーザー認証(Auth)、暗号化データの保管(Firestore)、および匿名統計収集(Analytics)のためのインフラを提供。
  • OpenAI: AI デイリーインサイトメッセージ生成のための人工知能エンジンを提供（送信されたデータはモデル学習には使用されません）。

サポートチーム: support@franholdings.cloud
