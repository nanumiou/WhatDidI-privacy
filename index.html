개인정보처리방침 (Privacy Policy)

최종 수정일: 2026년 4월 14일

1. 데이터 처리 원칙

"뭐했지 (WhatDidI)" 앱은 사용자의 프라이버시를 최우선으로 생각합니다. 본 서비스는 '로컬 우선(Local-first)' 철학에 따라, 하루를 기록하는 데 필요한 모든 원시 데이터를 사용자의 기기 내부에서만 처리하고 보관하는 것을 기본 원칙으로 합니다.

앱을 통해 수집된 위치 정보와 앱 사용 통계 등 민감한 데이터는 서버로 전송되지 않으며, 개발자조차 그 내용을 절대 열람할 수 없습니다.

2. 수집 및 활용하는 데이터 항목

앱의 주요 기능인 '자동 활동 기록'을 위해 다음과 같은 정보에 접근하고 처리합니다.

A. 기기 내 로컬 처리 데이터 (서버 전송 없음)

  • 위치 정보 (Background Location): 하루 이동 경로 및 머무른 장소를 파악하여 타임라인을 구성하기 위해 사용됩니다.
  • 앱 사용 통계 (Usage Stats): 특정 시간에 어떤 앱을 사용했는지 분석하여 활동(예: 공부, SNS, 영상 시청)을 추론합니다.
  • 활동 인식 (Activity Recognition): 사용자의 이동 상태(걷기, 차량 이동, 정지)를 감지하여 기록의 정확도를 높입니다.

B. 선택적 클라우드 동기화 데이터 (사용자 선택 시)

기기 변경이나 데이터 백업을 위해 사용자가 '클라우드 동기화'를 활성화할 경우에만 다음 데이터를 서버에 보관합니다.

  • 암호화된 활동 기록: 타임라인 데이터 및 사용자 설정 정보. 모든 데이터는 **군사 수준의 AES-256 암호화**를 거쳐 저장되며, 본인 외에는 누구도 해독할 수 없습니다.
  • 소셜 로그인 식별자: Google 계정을 통한 사용자 인증 정보 (본인 확인 및 데이터 소유권 검증용).

3. 특수 권한 사용 안내

본 앱은 자동 기록 기능을 위해 안드로이드 시스템의 다음 권한을 요청합니다.

  • 백그라운드 위치 액세스: 앱이 화면에 떠 있지 않을 때도 위치 변화를 감지하여 끊김 없는 타임라인을 구성하기 위해 필수적입니다.
  • 사용 정보 액세스: 앱별 사용 시간을 쿼리하여 현재 사용자가 무엇을 하고 있는지 지능적으로 추론하기 위해 사용됩니다.

4. 데이터 보관 및 파기

로컬 데이터는 앱 삭제 시 함께 제거됩니다. 클라우드 동기화를 사용 중인 경우, 앱 내 '설정 > 계정 관리' 메뉴의 [계정 삭제] 기능을 통해 서버에 보관된 모든 암호화 데이터를 즉시 영구 삭제할 수 있습니다.

5. 웹을 통한 계정 및 데이터 삭제 요청

앱을 이미 삭제했거나 앱에 접속할 수 없는 경우, 아래의 방법으로 데이터 삭제를 요청하실 수 있습니다.

  • 요청 방법: 지원 이메일로 가입 시 사용한 Google 계정 정보를 기재하여 발송
  • 이메일 주소: support@franholdings.cloud
  • 처리 기간: 접수 후 영업일 기준 7일 이내 데이터 파기 및 회신

6. 제3자 서비스 제공

안정적인 인증 및 데이터 보관을 위해 아래 외부 서비스를 활용합니다.

  • Google Firebase: 사용자 인증(Auth) 및 암호화된 데이터 보관(Firestore/Storage)을 위한 보안 인프라 제공.
